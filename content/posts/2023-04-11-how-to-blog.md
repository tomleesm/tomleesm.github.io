## 當作寫日記

每天寫一篇部落格文章，主題是今天做了什麼？學到什麼？即使只有一行字也好，當作寫日記。一天一篇，不多不少，如果想多寫，先放在別處，例如 Dropbox，但是不要放到部落格中，可以每天把之前的文章存檔 PO 出來。如果一開始就每天寫好幾篇還 PO 出來，壓力會很大，體力透支。

養成習慣需要的是頻率高，不是時間多。與其花好幾天寫一長篇文章，不如每天都寫一點短短的、分成幾集的文章，更能養成習慣。

## 小抄

基本原則是先有小抄，累積並整理後生出一篇文章。XDite 在他的部落格中提到

> - 所以首先要做的就是告訴自己，我寫不出完美的東西，我只需要貼 command 小抄就好。
> - 再來是習慣性在 code 每寫到一個段落時，就 commit 一次，把剛剛 implement 的想法寫在 commit note 裡
> - 步驟操作到一小段落，就把 console 的指令貼到 ticket 上
> - 踢到一個鐵板，就到 ticket 上哭么一次。不管是 exception log 或是詭異解法都好
> - 每找到一則參考資源，就把網址貼到 ticket 上去。避免之後手賤開了一堆窗，結果愚蠢的習慣性分頁全關滅團…
> - 確定完成該項工作後，「當天」找 10-20 分鐘整理工作細節。

這裡的 commit 是指 GIt commit，ticket 是指 Redmine issue。簡單來說就是把寫程式的過程，甚至是剛剛寫好的 code(注意著作權問題)，通通記錄到 issue tracking system，等工作完成後整理成筆記，放到 wiki 上，提供之後查詢。

或者是隨身帶著小筆記本，有什麼靈感先寫下來，一樣是寫小抄的概念，之後再整理成完整的文章

## 用紙筆手寫大綱和草稿

用紙筆手寫比起用電腦打字更容易促進思考[^1]，隨手打開筆記本寫字比用電腦更快速。想想看用電腦打字，需要哪些步驟？(我使用  Hugo 寫部落格)

1. 等電腦開機
2. 輸入作業系統登入密碼
3. 打開編輯器
4. 打開部落格目錄
5. 新增一個 markdown 檔案
6. 取個適當的標題
7. 切換到中文輸入法
8. 開始打字

到這個時候恐怕已經忘記要寫什麼了。用紙筆手寫，只需要：

1. 拿出筆記本
2. 翻到新的一頁
3. 手拿著筆開始寫

用電腦把草稿打字和補充，完成初稿

## 修改

文章和程式碼一樣，不可能一開始就寫的好，必須經過不停的修改、整理和「重構」，逐漸改善才行。總共有三個步驟，一開始放開來寫，想寫什麼就寫什麼。然後休息一下，喝杯水，讓自己清醒過來。接著修改文章，例如多餘的字、被動句或錯別字。第二步驟結束後就可以發佈文章了，因為要過幾天才會做第三階段，以一個外人的角度欣賞自己寫的文章

如何修改文章？好文章的標準是什麼？翻閱許多書籍，覺得「風格的要素」是唯一的答案，雖然是講英文寫作，但是大部分原則都適用於所有的文章
